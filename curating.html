Curating the Design Library

The CIF Design Library is intended to serve as a showcase and basic documentation for design modules. The file itself is a basic HTML document. When adding a module to the library, the module's name should be marked up as an <h2> and given a unique id attribute. The find-section-links.js script finds all <h2> elements with ids and creates section links to those headings in the sub-header. Modules should be separated with <hr /> elements.

All design modules should have a demo, with a button for displaying the necessary markup. This is easy enough to do; simply create a <div> with a unique id such as {module_name}-demo, place only the necessary markup for the module inside that wrapper, and add a button to the document with the following markup:

<a href="#" class="show-markup button" data-for="{module_name}-demo">Show Markup</a>

The show-markup.js script finds all elements with a show-markup class and displays the markup within the element with the same id as the button's data-for attribute when the button is clicked.

By default, the markup's container element is not included in the displayed markup. If the container should also be displayed, an additional include-container class can be added to the button.
